# このリポの現時点まとめ

このファイルは、別環境のChatGPTや将来の自分に対して、
「このリポは何を作ろうとしていて、今どこまで来ているのか」を短時間で共有するための要約です。

---

## 1. このリポは何か

これは `IDOL CROSSING` という、アイドル情報を横断して見られるサイト兼アプリのリポジトリです。  
単なるニュースサイトではなく、以下を1つの導線でつなぐことを狙っています。

- アイドルニュース
- イマキテランキング（日次 / 週次）
- ナンダッテ（投票参加型）
- バズッタラ（話題ポストのアーカイブ）
- 横断検索

一言でいえば、  
**「アイドル情報を、見るだけでなく、参加しながら追える場」にしようとしているリポ**です。

---

## 2. このプロジェクトに流れている思想

このリポには機能要件以上に、運営思想が入っています。現時点で強く見える軸は以下です。

### 2.1 情報を分散させたくない

アイドルを追うとき、ニュース、SNS、ランキング、ファンの熱量が別々の場所に散らばりやすい。  
このリポは、それらをなるべく1か所で接続し、
**「今このグループに何が起きているか」を立体的に見せたい**という思想で作られています。

### 2.2 見るだけで終わらせたくない

特に `ナンダッテ` に強く表れているように、
ファンは受け手であるだけでなく、文脈を作る側でもある、という考えがあります。

- 投票は単なる遊びではなく「推しの見え方を作る行為」
- ファンの言葉がグループの輪郭を形作る
- 参加ログそのものがサイト価値になる

つまり、
**推し活の情報消費を、推し活の参加・貢献体験に変えたい**という意図があります。

### 2.3 グループ単位で情報を束ねたい

将来構想では、ニュース / ランキング / SNS反響 / 投票を
すべて `Group` を中心に統合したい意志がかなり明確です。

再設計メモでも、最終的には
**「4サービスの寄せ集め」ではなく「グループ中心の情報OS」**
として再構成したい方向が示されています。

### 2.4 “ファンが自然に動ける導線”を重視している

よく出てくる理想の流れはこれです。

- 知る
- 見る
- 参加する
- 振り返る

ニュースで知り、ランキングで勢いを見て、バズを追い、投票で参加する。  
この一連の流れを自然につなげることが、かなり中核にあります。

## 3. 今あるサービス構成

### 3.1 ホーム

トップページは各サービスの更新をまとめて見せるハブです。  
`イマキテ / ナンダッテ / バズッタラ / ニュース / 関連イベント` の要約を並べる方向で実装されています。

### 3.2 アイドルニュース

- 既存WordPress記事とSanity記事を統合表示
- カテゴリ / タグ絞り込みあり
- 将来的にはSanity中心運用へ寄せる想定

現状は、**WPをまだ活かしながらNext.js側で表示を統合する段階移行**が前提です。

### 3.3 イマキテランキング

- 日次Top20
- 週次Top20
- 過去アーカイブ
- Spotify埋め込み付きランキング表示

「今来てるアイドル」を、日次・週次で分かりやすく見る導線です。

### 3.4 ナンダッテ

このリポの思想が最も濃く出ている機能です。

- グループごとの魅力語彙を投票で可視化する
- 600組以上のグループを対象に個別ページを持たせる想定
- 「みんなにどう思われているか」を魅力Top5などで見せる
- 自分の投票履歴も価値として持たせる

単なる人気投票ではなく、
**ファンがグループの“見え方”を共作する場**として考えられています。

### 3.5 バズッタラ

- 100万View以上の話題ポストをアーカイブ
- 数字やタグを通して話題の背景も見せる

SNS上の熱量を、ニュースやランキングと別軸のリアルタイム指標として扱っています。

### 3.6 横断検索

- ニュース
- ナンダッテ
- バズッタラ

をまたいで検索する入口。  
現行でも重要ですが、将来的にはさらに中核導線にしたい意図が見えます。

## 4. 技術的な現状

### 4.1 フロントエンド / アプリ

- Next.js App Router
- React
- TypeScript
- Tailwind CSS v4

### 4.2 CMS / データ基盤

- Sanity
- Supabase
- 既存WordPress（ヘッドレス的に利用）

構造としては、
**Next.jsを表示基盤にしつつ、WP / Sanity / Supabase を段階的に整理統合している状態**です。

### 4.3 ニュースの扱い

ニュースは現状、

- 既存記事はWordPress由来
- 新規記事はSanityで作る方向
- 旧記事は優先度順にSanityへ移行

という段階移行方針です。

要するに、
**UI刷新を先に進めながら、運用の重心をWordPressからSanityへ移している途中**です。

### 4.4 Supabaseの役割

Supabaseは特に以下の文脈で使われています。

- ナンダッテの投票データ
- イマキテのランキングデータ
- 認証
- 将来的な検索 / 集計 / read model の受け皿

このリポは、単なるCMSフロントではなく、
**参加データや集計データを持つアプリ寄りのメディア**です。

## 5. 現時点での重要な判断・前提

### 5.1 WordPressはすぐ捨てない

既存記事が大量にあるため、一括移行はしない。  
まずはヘッドレス的に取り込み、段階的にSanityへ寄せる方針です。

### 5.2 Sanityは今後の新規記事運用の中心候補

編集者が1人であること、構造化した運用を強めたいことから、
Sanityの相性が良いという判断が入っています。

### 5.3 ナンダッテは単独機能ではなく全体の核になりうる

要件書や再設計メモを見る限り、
ナンダッテは「参加型サブ機能」ではなく、
将来的にはサイト全体の差別化コアとして育てたい意志があります。

### 5.4 今は現行改善とゼロベース再設計を分けて考えている

すぐ効く改善は現行トラックで回しつつ、
別ラインでより大きな再設計構想も温めています。  
つまり「目先の改善」と「思想レベルの再構築」が並行しています。

## 7. まだ構想段階だが重要なもの

### 7.1 カイワイ

ナンダッテの投票共起から、
「同じユーザーに投票されやすいグループの近さ」を可視化する構想です。

これは単なるレコメンドではなく、
**ファン行動から界隈感・文脈の近さを浮かび上がらせる試み**です。

### 7.2 Group Hub / Group Timeline 的な再編

再設計メモでは、
グループ単位でニュース・ランキング・SNS・投票を時系列統合する方向が明確です。  
この思想は、このリポの将来像を理解する上でかなり重要です。

### 7.3 参加履歴・しおり・通知などのMy Page化

今後はユーザーの行動ログ自体を価値化し、
「見るだけでなく積み上がる体験」に寄せたい流れがあります。

## 9. ChatGPTに共有するときの短い説明文

必要なら、以下の説明をそのまま渡してよいです。

このリポは、IDOL CROSSING というアイドル情報サイト兼アプリのコードベースです。  
目的は、ニュース・ランキング・SNS話題・ファン投票を1つの導線でつなぎ、ユーザーが「知る→見る→参加する」を自然に行える体験を作ることです。  
特にナンダッテは、ファン投票によってグループの魅力語彙や見え方を可視化する中核機能として位置づけられています。  
技術的には Next.js / Supabase / Sanity / 既存WordPress の併用構成で、ニュースはWPからSanityへ段階移行中です。  
将来的には、各サービスをバラバラに運用するのではなく、Group中心にニュース・ランキング・SNS・投票を束ねた統合体験へ再設計したい意図があります。

という思想をかなり強く持っています。

そのため、

- メディア
- データ基盤
- 参加型アプリ

の3つが混ざったような性格を持っています。  
この混ざり方こそが、現時点のこのリポの本質です。
