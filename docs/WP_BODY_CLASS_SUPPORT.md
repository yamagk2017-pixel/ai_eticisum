# WP本文クラス対応メモ（ヘッドレスWP / 暫定表示）

## 目的

WordPress本文（`content.rendered`）に含まれる独自クラスのうち、ヘッドレス表示で崩れやすいものを整理する。

このドキュメントは以下を目的とする。

- 現在サポートしているWP本文クラスの明文化
- 各クラスの意味（WP側の運用意図）の記録
- Next.js側での暫定再現方針の共有
- 今後のUI調整時の追記先（運用メモ）

## 前提（重要）

- `完全再現は目指さない`
- 目的は `WP記事を表示しつつ、将来的にSanityへ移植していくこと`
- 現在は `仮状態の表示` でよい（意味が伝わることを優先）
- 今後もヘッドレスWP記事のUI調整は継続予定
- 調整のたびにこのファイルへ追記・更新する

## 実装場所（現時点）

- 本文表示コンポーネント: `src/components/news/wp-article-body.tsx`
- WP HTML整形（style除去など）: `src/lib/wp/client.ts`

## 対応方針（基本ルール）

1. `意味のあるクラス` は活かす
- 例: リード文、右寄せ画像、キャプションなど

2. `見た目の完全一致` ではなく `役割の再現` を優先
- WPテーマ依存の細かい装飾は再現しない
- ただし読解性/構造が崩れるものは対応する

3. `未対応クラス` は基本そのまま通す
- まず表示確認
- 問題が出た時だけ追加対応

4. `WP由来のinline styleは原則除去`
- テーマや端末差で崩れやすいため
- Next側の表示ルールを優先する

## サポート中のWP本文クラス（現時点）

### `well3`

- WP側の意味:
  - リード文/強調ブロック用の汎用クラス
  - 他の段落との差別化（罫線 + 背景色）

- Next側の扱い（暫定）:
  - 背景色あり
  - 枠線あり
  - 余白あり
  - 本文の可読性を優先した簡易ボックス表示

- 備考:
  - 「WPと完全に同じ見た目」ではなく、意味が分かることを優先

### `alignright`

- WP側の意味:
  - 画像/ブロックの右寄せ

- 確認済みパターン:
  - `img.alignright`
  - `div.alignright`（`wp-caption` と併用されるケースあり）

- Next側の扱い（暫定）:
  - 右寄せ（`float-right` ベース）
  - 左余白/下余白を付与
  - 幅を制限（本文を壊しにくくする）
  - 親段落のレイアウト崩れを抑制

- 備考:
  - スマホでも右寄せ有効（現方針）
  - 将来的に必要ならモバイルのみ解除の検討余地あり

### `wp-caption`

- WP側の意味:
  - 画像 + キャプションを囲む枠

- Next側の扱い（暫定）:
  - 枠線あり
  - 背景色あり
  - 内側余白あり
  - 角丸なし（現在方針）

- 備考:
  - `alignright` と併用されるケースを想定済み

### `wp-caption-text`

- WP側の意味:
  - キャプション本文

- Next側の扱い（暫定）:
  - 小さめ文字
  - 補助色
  - 行間調整
  - キャプション下の余白は削減（段落マージン打ち消し）

## HTML整形（クラス以外）

### 除去しているもの（`src/lib/wp/client.ts`）

- `<style>...</style>`
- `style=""`（inline style）
- `data-mce-style`
- `color=""`
- `bgcolor=""`

理由:
- WPテーマ/エディタ由来の見た目指定が強すぎると、ヘッドレス側のUI設計を壊しやすいため

## 画像リンクの拡大表示（関連仕様）

本文クラスではないが、本文表示互換の一部として記録。

- 実装: `src/components/news/wp-article-body.tsx`
- 方針:
  - `a > img` かつ `href` が画像URLなら `dialog` モーダルで拡大表示
  - スマホの拡大操作はブラウザ任せ
  - 画像URLの厳密一致は見ない（MVP方針）

## 非サポート / 未確認（今後追加）

以下は未確認または未対応。必要時に追記する。

- `alignleft`
- `aligncenter`
- `size-*` 系クラス
- `wp-image-*`
- ギャラリープラグイン固有クラス（要記事実例）
- 音声/動画埋め込み系のWPクラス

## 新しいWPクラスを見つけた時の運用手順

1. `対象記事ID` をメモする
2. 実DOMを確認（どのタグに class が付いているか）
3. WP側での意味（編集意図）を確認する
4. Next側での再現方針を決める
- 完全再現ではなく役割再現を優先
5. `src/components/news/wp-article-body.tsx` に追加実装
6. このファイルに追記（クラス名、意味、扱い）
